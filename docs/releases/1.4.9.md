# CourseForge 1.4.9

## Summary

CourseForge 1.4.9 hardens cloud OCR diagnostics and execution order. Cloud OCR is now explicitly cloud-first, local-last, and each cloud failure path reports exact provider-specific telemetry instead of a generic fallback message.

## Included Changes

- Added structured OCR diagnostics for OpenAI Vision and GitHub Models Vision, including trace ID, reason code, HTTP status, and failure stage.
- Added provider-specific backend health probes through the shared Firebase callable path.
- Added real GitHub Models OCR extraction support in the deployed backend and frontend selection flow.
- Updated the Settings UI to expose primary and secondary cloud providers while keeping local Tesseract as the final fallback.
- Added `npm run test:smoke:ocr:cloud` to generate a representative textbook-style image and validate both cloud OCR providers with a JSON report.
- Updated updater integration coverage so `/releases/latest?cache_bust=...` is accepted during manual and automatic update checks.

## Validation

- `npm run test:e2e`
- `npm run test:smoke:ocr:cloud`

## Smoke Result

Latest verified smoke outcome on this release candidate:

- `cloud_github_models_vision`: success, HTTP 200, usable OCR text returned
- `cloud_openai_vision`: failure, HTTP 429, `rate_limited`, provider message indicates quota/billing exhaustion

OpenAI is still instrumented and attempted first when configured, but 429 responses are now classified as unavailable so CourseForge can move to the secondary cloud provider instead of treating the provider as healthy.
